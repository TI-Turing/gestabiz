# Sistema de Internacionalización (i18n)

~2,200 claves de traducción type-safe en español e inglés distribuidas en ~44 archivos modulares por idioma.

## Descripción

Sistema i18n modular con auto-completado TypeScript. Español como idioma default. Cada dominio tiene su archivo de traducciones.

## Estructura

```
src/locales/
├── es/          # ~44 archivos en español
│   ├── absences.ts
│   ├── admin.ts
│   ├── adminDashboard.ts
│   ├── appointments.ts
│   ├── auth.ts
│   ├── business.ts
│   ├── jobs.ts
│   └── ... (~37 más)
├── en/          # ~44 archivos en inglés (mirror)
├── index.ts     # Re-exports
└── types.ts     # Tipos TypeScript
```

## Uso

```tsx
const { t } = useLanguage();

// Clave simple
t('dashboard.title')

// Clave con parámetros
t('appointments.count', { count: 5 })

// Formato de moneda
formatCurrency(amount, 'COP', 'es')
```

## Contexto

`LanguageProvider` en `src/contexts/LanguageContext.tsx`:
- Expone `t(key, params)` para traducciones
- Utilidades de formato (fecha, moneda, números)
- Persistencia del idioma seleccionado en localStorage vía `useKV`

## Legacy

`src/lib/translations.ts` sigue existiendo por compatibilidad. Las traducciones de locales/ y translations.ts se mergean (locales/ tiene prioridad).

## Type-Safe

Las claves están tipadas → auto-completado en el IDE. Error de compilación si se usa una clave que no existe.

## Archivos Clave

- `src/locales/` — Sistema modular
- `src/contexts/LanguageContext.tsx` — Provider
- `src/lib/translations.ts` — Legacy (compatibilidad)
