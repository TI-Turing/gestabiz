## 3. Technical SEO for React SPA (Vite)

### 3.1 El Problema de las SPAs con SEO

Google usa Chromium para renderizar JavaScript, pero en dos fases:
1. **Crawl**: descarga el HTML inicial (casi vacío en una SPA)
2. **Render**: ejecuta JS en una cola separada (puede tardar días/semanas)
3. **Index**: indexa el contenido renderizado

**Timeout de Googlebot**: ~5 segundos para scripts. Si el JS tarda más, timeout.

### 3.2 Estrategias de Rendering Recomendadas

#### Opción A: Pre-rendering en Build Time (RECOMENDADA para Gestabiz)
- Usar `vite-plugin-prerender` o `vite-ssg`
- Genera HTML estático para rutas públicas en `vite build`
- Ideal para: Landing page, /negocio/:slug, blog posts
- **Ventaja**: Sin cambio de framework, compatible con Vercel
- **Implementación**: Pre-renderizar solo rutas públicas, mantener CSR para /app/*

#### Opción B: Prerender.io / Rendertron (middleware)
- Servicio externo que cachea HTML renderizado
- Detecta bots (Googlebot, Bingbot) y sirve HTML estático
- Mantiene la SPA intacta para usuarios normales
- **Costo**: Prerender.io desde $9.99/mes

#### Opción C: SSR parcial con React Router 7+ o Remix
- Migración más costosa pero mayor control
- SSR para rutas públicas, CSR para dashboard autenticado
- **No recomendado ahora** — el codebase es grande y estable

#### Opción D: Edge SSR con Vercel (React Server Components)
- Requiere migración a Next.js o framework con RSC
- **No recomendado** — demasiado disruptivo para el estado actual

### 3.3 Implementación Recomendada para Gestabiz

**Fase 1: Pre-rendering de rutas públicas**
```
Rutas a pre-renderizar:
- / (Landing page)
- /login
- /register
- /negocio/:slug (perfiles públicos — generar lista dinámica)
- /blog/* (futuro)
```

**Fase 2: Meta tags dinámicos**
- Ya implementado parcialmente con `usePageMeta`
- Asegurar que cada ruta pública tenga title, description, og:image únicos
- Usar `react-helmet-async` para SSR-compatible meta management

**Fase 3: Sitemap dinámico**
- Ya existe `npm run generate-sitemap`
- Automatizar generación en cada deploy
- Incluir todos los `/negocio/:slug` con lastmod

### 3.4 Core Web Vitals Optimization

**Métricas target (2026)**:
- **LCP** (Largest Contentful Paint): < 2.5s
- **INP** (Interaction to Next Paint): < 200ms (reemplazó a FID en 2024)
- **CLS** (Cumulative Layout Shift): < 0.1

**Acciones específicas para Gestabiz**:

1. **LCP**:
   - `<link rel="preload">` para hero image de landing
   - Lazy load de imágenes below-the-fold
   - Optimizar bundle size (ya usa code splitting)
   - Servir imágenes en WebP/AVIF desde Supabase Storage

2. **INP**:
   - `startTransition` para actualizaciones no urgentes
   - Debounce en SearchBar (ya implementado: 300ms)
   - Evitar re-renders innecesarios con React.memo

3. **CLS**:
   - Dimensiones explícitas en todas las imágenes
   - Skeleton loaders (ya parcialmente implementado)
   - `font-display: swap` para custom fonts
   - Reservar espacio para contenido dinámico
